# 하루 소비 내역을 성격 캐릭터로 읽어주는 앱 PRD

## 1. 문서 개요

- 제품명: `오늘의 소비 캐릭터`
- 제품 한 줄 설명: 하루 지출 내역을 읽고 `오늘 나는 어떤 소비 성향이었는지`를 별명, 태그, 한 줄 해석으로 보여주는 서비스
- 문서 목적: MVP 기준의 제품 요구사항, 핵심 UX, 입력/출력 구조, 결과 톤을 정의해 빠르게 검증 가능한 기준 문서로 사용한다

---

## 2. 배경

사람들은 카드 내역과 계좌 알림을 자주 보지만,
그 기록만으로는 `오늘 내가 왜 이렇게 썼는지`를 감각적으로 이해하기 어렵다.

기존 가계부는 숫자와 합계 중심이라 실용적이지만,
사용자가 자기 소비를 `성격처럼` 받아들이고 기억하게 만들지는 못한다.
반대로 MBTI나 심리 테스트형 콘텐츠는 결과가 기억에 남고 공유되지만,
실제 행동 데이터와 연결되어 있지 않은 경우가 많다.

지금 필요한 건
딱딱한 지출 리포트도 아니고
뜬구름 잡는 성격 테스트도 아니다.

이 제품은 하루 소비 데이터를 바탕으로
`오늘의 소비 캐릭터`를 만들어
사용자가 자기 지출을 더 빨리 이해하고,
가볍게 웃으면서도 다음 행동 힌트를 얻도록 돕는다.

---

## 3. 제품 비전

이 제품의 비전은
소비 기록을 죄책감의 근거가 아니라
`나를 읽는 재치 있는 거울`로 바꾸는 것이다.

초기 MVP는 하루 단위 소비 캐릭터 판정에 집중한다.
장기적으로는 주간/월간 성향 변화,
반복 소비 습관,
친구와 비교 가능한 공유형 캐릭터 리포트까지 확장할 수 있다.

### 3.1 Why Now

- LLM 기반 해석으로 불규칙한 카드/계좌 알림 문구도 자연어 수준으로 읽어낼 수 있다
- 사용자는 숫자 리포트보다 `한 줄 정체성 요약`에 더 빠르게 반응하고 공유한다
- 셀프 트래킹 시장에서 `실용성 + 밈성`을 함께 가진 소비 경험은 아직 드물다

---

## 4. 목표

### 4.1 제품 목표

1. 소비 내역을 `오늘의 소비 캐릭터`라는 기억 가능한 결과로 요약한다.
2. 사용자가 하루 소비 패턴을 10초 안에 이해하게 한다.
3. 결과가 웃기기만 한 것이 아니라 `왜 이런 캐릭터가 나왔는지`를 납득하게 한다.
4. 사용자가 다음 소비에서 의식할 한 가지 행동 포인트를 얻도록 한다.

### 4.2 UX 목표

1. 입력은 붙여넣기 수준으로 단순해야 한다.
2. 첫 화면에서 캐릭터명과 한 줄 해석이 바로 보여야 한다.
3. 결과는 스크린샷을 찍고 싶을 정도로 개성 있어야 한다.
4. 결과는 놀리기보다 `정확하게 나를 본 느낌`이어야 한다.

### 4.3 비목표

- 정밀한 자산 관리 서비스
- 은행 연동 기반 자동 가계부
- 투자 자문
- 세무/회계 목적의 리포트
- 임상적 성격 진단처럼 보이는 경험

---

## 5. 대상 사용자

### 5.1 핵심 사용자

- 가계부는 귀찮지만 자기 소비 성향은 궁금한 사용자
- 숫자 표보다 한 줄 해석과 태그형 피드백을 선호하는 사용자
- 자신의 생활 패턴을 재미있게 돌아보고 싶은 사용자
- 공유 가능한 결과물을 좋아하는 사용자

### 5.2 주요 페르소나

#### 페르소나 A: 퇴근 후 소비를 자주 하는 직장인

- 특성: 평일 소비가 피로 회복용으로 몰리는 편
- 문제: 쓴 금액보다 `왜 이런 소비가 몰렸는지`가 더 궁금함
- 기대: 오늘 소비가 보상형이었는지 루틴형이었는지 금방 알고 싶음

#### 페르소나 B: 자잘한 소비가 많은 학생/사회초년생

- 특성: 편의점, 배달, 카페 등 소액 지출이 잦음
- 문제: 새는 돈이 있다는 건 아는데 패턴 설명이 잘 안 됨
- 기대: 지출 흐름을 캐릭터처럼 쉽게 받아들이고 싶음

#### 페르소나 C: 밈성과 공유 포인트에 반응하는 사용자

- 특성: 테스트 결과, 카드형 리포트, 짧은 별명을 좋아함
- 문제: 금융 앱은 유용하지만 재미가 없어서 다시 열지 않음
- 기대: 결과를 보고 `이거 나잖아` 하고 저장하거나 공유하고 싶음

---

## 6. 핵심 문제

사용자는 아래 문제를 겪는다.

- 거래 내역이 있어도 그날 소비의 `캐릭터`가 보이지 않는다
- 작은 소비의 누적 흐름을 기억에 남는 방식으로 이해하기 어렵다
- 숫자 위주 리포트는 금방 닫게 된다
- 반성은 해도 다음 행동으로 이어질 만큼 선명한 해석이 남지 않는다

이 제품은 아래 질문에 답해야 한다.

- 오늘 나는 어떤 소비 캐릭터였는가?
- 그 캐릭터를 만든 핵심 지출은 무엇이었는가?
- 이 결과가 왜 그렇게 나왔는가?
- 내일 균형을 잡으려면 무엇을 의식하면 되는가?

---

## 7. 핵심 가치 제안

이 제품은 단순한 소비 요약기가 아니라
`소비 데이터를 성격 캐릭터처럼 읽어주는 해석 엔진`이다.

핵심 가치는 다음 4가지를 동시에 준다.

1. 숫자를 정체성 언어로 바꿔 기억에 남게 한다
2. 결과가 재치 있어서 저장/공유 동기를 만든다
3. 캐릭터 판정 근거를 함께 보여줘 납득감을 준다
4. 가벼운 행동 힌트로 실용성을 놓치지 않는다

---

## 8. 핵심 서비스 컨셉

### 8.1 결과 구조

결과는 아래 순서로 노출한다.

1. `오늘의 소비 캐릭터명`
2. `한 줄 성격 요약`
3. `캐릭터가 나온 근거 소비 2~3개`
4. `보조 태그 3개`
5. `내일의 한 수`

### 8.2 캐릭터 네이밍 원칙

캐릭터명은 MBTI 결과처럼 기억되고,
사용자를 평가하지 않으면서도 살짝 웃기게 만든다.

네이밍 패턴 예시:

- `[생활 장면] + [역할/인물]`
- `[욕구/감정] + [행동가]`
- `[소비 습관] + [캐릭터명]`

예시:

- 야식 앞에서 한없이 다정한 `편의점 순례자`
- 퇴근 후 나를 챙기는 `셀프 포상러`
- 할인 알림에 민첩한 `쿠폰 사냥꾼`
- 이동과 카페인이 하루를 끈 `도시 질주가`
- 소소하게 자주 쓰는 `잔지출 수집가`

### 8.3 실용성 유지 장치

재미 요소만 남기지 않기 위해
모든 캐릭터 결과에는 반드시 아래가 따라와야 한다.

- 근거 거래 2~3개
- 핵심 소비 패턴 1줄
- 내일 행동 힌트 1개
- 추정성 문구 표시

---

## 9. MVP 범위

### 9.1 포함 범위

- 카드/계좌 알림 텍스트 붙여넣기
- 날짜별 소비 캐릭터 1개 생성
- 카테고리 추정
- 캐릭터명 1개 + 보조 태그 3개 생성
- 캐릭터 판정 근거 소비 항목 표시
- 내일의 한 수 1개 제안
- 저장/공유 가능한 결과 카드

### 9.2 제외 범위

- 은행 계정 직접 연동
- 월간 예산 설정
- 자동 푸시 알림
- 가족 공동 가계부
- 친구 랭킹/피드 기능
- 정교한 심리 검사형 질문지

## 10. 입력/출력 정의

### 10.1 입력

- 하루 소비 내역 텍스트
- 선택 입력: 오늘 기분 또는 상황 메모
- 선택 입력: 사용자 설정 예산

### 10.2 출력

- 오늘의 소비 캐릭터명
- 한 줄 성격 요약
- 보조 태그 3개
- 근거가 된 주요 소비 항목 2~3개
- 소비 패턴 해석 1줄
- 내일의 한 수
- 저장/공유용 카드형 결과

### 10.3 출력 예시

#### 예시 A

- 캐릭터명: `퇴근 후 보상 챙기는 셀프 포상러`
- 한 줄 요약: 오늘 소비는 생존보다 위로에 가까웠어요. 피곤한 하루 끝에 작은 보상을 여러 번 챙긴 날입니다.
- 태그: 보상형, 카페인형, 소액반복형
- 근거 소비: 저녁 카페, 배달 디저트, 편의점 간식
- 내일의 한 수: 보상 소비는 하나만 남기고 나머지는 미리 정한 예산 안에서 고르기

#### 예시 B

- 캐릭터명: `시간을 돈으로 산 편의형 해결사`
- 한 줄 요약: 오늘은 절약보다 빠른 해결이 중요했던 날이에요. 이동과 간편 결제가 몰리며 편의 우선 성향이 강했습니다.
- 태그: 편의형, 이동형, 즉시해결형
- 근거 소비: 택시, 배달, 편의점
- 내일의 한 수: 내일 한 번만 느린 선택을 해보면 편의 비용 패턴이 더 선명해집니다

---

## 11. 핵심 사용자 흐름

### 11.1 기본 플로우

1. 사용자가 소비 알림 또는 내역을 붙여넣는다.
2. 필요 시 오늘 상황 메모를 추가한다.
3. 생성 버튼을 누른다.
4. 결과 화면에서 `오늘의 소비 캐릭터`와 근거를 확인한다.
5. 결과를 저장하거나 공유한다.

### 11.2 빠른 사용 플로우

1. 사용자가 소비 내역만 붙여넣는다.
2. 기본 설정으로 캐릭터를 생성한다.
3. 캐릭터명, 한 줄 요약, 내일의 한 수만 빠르게 확인한다.

---

## 12. 정보 구조

제품은 3개의 핵심 화면으로 구성한다.

1. 입력 화면
2. 결과 화면
3. 최근 캐릭터 화면

MVP에서는 히스토리 기능이 얕아도 괜찮지만,
`어제 나는 어떤 캐릭터였는지` 다시 보는 진입은 필요하다.

---

## 13. 화면 요구사항

### 13.1 입력 화면

#### 목적

사용자가 소비 기록을 가장 적은 마찰로 넣고
결과가 어떤 톤인지 미리 감 잡게 한다.

#### 필수 요소

- 제품명
- 소비 내역 입력 영역
- 예시 포맷
- 선택 메모 입력
- 생성 버튼
- 결과 예시 프리뷰

#### UX 요구사항

- 붙여넣기 중심 UX여야 한다
- 입력 형식이 완벽하지 않아도 받아들여야 한다
- 예시는 실제 카드 문자처럼 보여야 한다
- 생성 전부터 `이 앱은 분석보다 캐릭터를 뽑아준다`는 기대가 전달돼야 한다

### 13.2 결과 화면

#### 목적

숫자보다 `오늘 나는 어떤 소비 성향이었는가`가 먼저 읽히는 리포트를 보여준다.

#### 필수 요소

- 오늘의 소비 캐릭터 Hero
- 한 줄 성격 요약
- 보조 태그
- 근거 소비 카드
- 내일의 한 수
- 저장/공유 버튼
- 다시 생성 버튼

#### UX 요구사항

- 첫 화면에서 캐릭터명과 요약이 바로 보여야 한다
- 결과는 재치 있지만 비꼬지 않아야 한다
- 사용자가 `왜 이 결과가 나왔는지` 바로 납득할 수 있어야 한다
- 리포트 전체 길이는 짧고 스캔 가능해야 한다

## 14. 컴포넌트 요구사항

### 14.1 Transaction Input Box

- 멀티라인 붙여넣기 입력
- 불규칙한 포맷 허용

### 14.2 Character Hero

- 오늘의 소비 캐릭터명
- 한 줄 성격 요약
- 가장 강한 소비 분위기 전달

### 14.3 Tag Chips

- 보조 태그 3개
- 예: 보상형, 편의형, 잔지출형, 이동형, 루틴형

### 14.4 Evidence Card

- 캐릭터 판정의 근거가 된 지출 항목
- 금액 또는 빈도
- 짧은 해석 코멘트

### 14.5 Action Hint Block

- 내일 주의할 소비 포인트
- 지나치게 훈계조가 되지 않게 구성

### 14.6 Share Card

- 캐릭터명과 대표 문구를 보기 좋게 압축
- 저장/공유에 적합한 카드형 레이아웃

## 15. UI 원칙

### 15.1 제품 톤

- 가볍지만 얕지 않게
- 재치 있지만 사람을 놀리지 않게
- 판단보다 관찰에 가깝게
- 죄책감보다 자기이해를 우선

### 15.2 레이아웃 원칙

- 결과의 첫 화면은 캐릭터명과 한 줄 요약이 차지한다
- 복잡한 표보다 카드 중심 구조를 사용한다
- 모바일에서 1스크린 내 핵심 결과를 본다

### 15.3 카피 원칙

- `당신은 과소비형입니다`처럼 단정하지 않는다
- `오늘은 빠른 해결을 우선한 편의형 소비가 몰렸어요`처럼 관찰형 문장을 쓴다
- 별명은 웃기되 모욕적이지 않아야 한다
- 결과는 캡처하고 싶은 문장이어야 한다

### 15.4 시각 원칙

- 차가운 금융 앱보다 캐릭터 카드에 가까운 인상
- 과한 유아화는 피하고 감도 있는 위트 유지
- 컬러는 캐릭터 톤을 돕되 해석 정보보다 튀지 않게 사용

## 16. 상태 설계

### 16.1 입력 전 상태

- 예시 입력 제공
- 결과 예시 프리뷰 제공
- 생성 버튼 비활성

### 16.2 입력 중 상태

- 최소 길이 충족 시 버튼 활성화
- 포맷 에러보다는 파싱 가능 여부를 유연하게 처리

### 16.3 생성 중 상태

- 입력값 유지
- 결과 영역 skeleton 노출
- `오늘의 소비 캐릭터를 읽는 중` 같은 로딩 카피 사용 가능

### 16.4 결과 상태

- 캐릭터, 요약, 근거 소비, 힌트 노출
- 저장/공유 가능

### 16.5 에러 상태

- 거래 내역이 너무 적은 경우
- 형식을 읽지 못한 경우
- 재시도 예시 제공
- `캐릭터를 만들기엔 재료가 조금 부족해요` 같은 완충형 카피 사용

---

## 17. 콘텐츠 규칙

- 과도한 도덕 판단 금지
- 병리적 표현 금지
- 캐릭터명은 애정 어린 관찰 느낌이어야 한다
- 같은 패턴이라도 문구 반복률이 높지 않게 변주한다
- 근거 없는 추정은 하지 않고 추정은 추정으로 표시한다
- 사용자를 깎아내리는 유머보다 `정확해서 웃긴 유머`를 지향한다

---

## 18. 성공 지표

### 18.1 정성 지표

- `이거 진짜 나 같다`
- `가계부보다 훨씬 다시 보고 싶다`
- `웃기긴 한데 의외로 도움 된다`
- `친구한테 보여주고 싶다`

### 18.2 제품 지표

- 생성 완료율
- 결과 저장/공유 비율
- 결과 화면 체류 시간
- 7일 내 반복 사용률
- 캐릭터 재생성률

---

## 19. 리스크

- 결과가 억지 밈처럼 느껴질 수 있음
- 캐릭터명이 반복되면 금방 질릴 수 있음
- 해석이 과하게 단정적으로 느껴질 수 있음
- 거래 내역 포맷이 제각각이라 파싱 실패 가능성이 있음

완화 방향:

- 캐릭터명만 재치 있게 하고 근거 설명은 명확하게 유지한다
- 캐릭터 네이밍 라이브러리를 카테고리별로 충분히 확보한다
- 모든 결과에 `왜 이렇게 나왔는지` 근거를 붙인다
- 파싱 실패 시 예시 포맷을 즉시 제공한다

---

## 20. 향후 확장

- 주간 소비 캐릭터 변화 리포트
- 월급 직후/월말 소비 성향 비교
- 친구와 공유하는 캐릭터 카드
- 반복 결제 및 구독 성향 탐지
- `내 소비는 어떤 욕구에 반응하는가` 같은 상위 성향 분석
