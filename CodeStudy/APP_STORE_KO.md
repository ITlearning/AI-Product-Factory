## 설명 (4000자 제한)

```
ChatGPT는 답을 준다. 한 시간 후엔 잊힌다.
유튜브 강의는 듣고 나면 머리에 남지 않는다.
CodeStudy는 답을 만들게 한다. 그래서 기억에 남는다.

CodeStudy는 소크라테스식 방법으로 가르치는 AI 코딩 튜터입니다.
코드를 던져주는 대신, 당신이 직접 답을 찾을 때까지 질문합니다.
"아!" 하는 깨달음의 순간이 AI의 것이 아니라 당신의 것이 됩니다. 그래서 잊히지 않습니다.

— 두 트랙 중에서 선택 —

• Swift / iOS — 앱 개발자용
• Spring / Kotlin — 백엔드 개발자용

각 트랙마다 50개의 손으로 큐레이션한 핵심 개념과 같은 소크라테스식 대화 엔진. 설정에서 언제든 트랙 전환 가능.

— 어떻게 작동하나요? —

개념(옵셔널, SwiftUI 상태, Kotlin null safety, JPA 트랜잭션…)을 하나 골라 튜터와 대화를 시작합니다.
튜터는 당신의 이해도를 탐색하고, 비유를 들어 설명하고, 코드의 동작을 예측해보라고 합니다.
정말로 이해했다고 판단되면 그제야 "마스터" 표시를 줍니다.

객관식 없습니다. 복붙 없습니다. "잘했어요!" 같은 빈말 없습니다.
진짜로 깨달을 때까지 진심으로 주고받는 대화뿐입니다.

— 각 트랙엔 어떤 개념이 들어있나요? —

Swift / iOS (50개)
변수, 옵셔널, 클로저, 프로토콜, 제네릭, async/await, SwiftUI 상태, Concurrency, 매크로 등.

Spring / Kotlin (50개)
val/var, null safety, 데이터 클래스, 코루틴, Spring Boot DI, JPA, 트랜잭션, REST 설계, 대규모 시스템 패턴 등.

— 주요 기능 —

• 두 트랙 합 100개 이상의 개념 (트랙당 50개)
• 매일의 학습 리듬과 스트릭 트래킹
• 스트릭 보호 — 하루 빼먹어도 자동으로 지켜져요
• 트랙별 학습 기록 — 마스터한 개념의 모든 세션과 대화를 다시 펼쳐볼 수 있어요
• 진짜 의미있는 마스터리 (단순히 "한 번 봤음" 아님)
• 한국어/영어 지원
• 회원가입 없는 익명 학습 — 이메일·계정 안 받습니다
• 투명한 프라이버시 — 광고 트래커 0개

— 누구를 위한 앱인가요? —

진짜로 Swift를 배우고 싶은 iOS 개발자.
Spring/Kotlin으로 서버를 짓는 백엔드 개발자.
부트캠프 졸업 후 개념 빈틈을 메우고 싶은 분.
같은 부분에서 자꾸 막히는 독학 코더.
다음 주면 잊는 튜토리얼에 지친 분들.

— CodeStudy에 없는 것 —

광고 없음. 트래커 없음. 분석 SDK 없음. 결제 유도 없음.
그냥 튜터 하나뿐입니다.

— 만든 사람 —

빠른 답을 주는 데 최적화된 요즘 코딩 도구가 답답했던 1인 개발자가 만들었습니다.
CodeStudy는 진짜 중요한 것 — 장기 기억 — 에 최적화돼 있습니다.

지금 써보세요. 트랙 고르고 5분이면 첫 개념을 마스터할 수 있어요.
대화 자체가 학습입니다.
```

## 앱 개인정보 보호 세부사항 (App Store Connect → "앱 개인정보 보호")

> ✅ **1.2.0 — 변경 없음**. Cycle 3에서 추가된 `track` 필드는 새 데이터 종류가 아닌, 기존 "제품 상호 작용" 카테고리(개념 학습 진행도)에 포함됨. 4개 카테고리 그대로 유지.
>
> ⚠️ **1.0.x에서 "데이터가 수집되지 않음"으로 입력돼 있다면 반드시 변경.**
> 사실 1.0.x도 console.log로 Vercel logs에 흘렀지만, Neon Postgres 도입으로 명백한 "수집"이 됐음.
> Apple App Privacy 가이드 위반으로 reject 위험.

### 변경 절차

App Store Connect → 앱 정보 → "앱 개인정보 보호" → **"앱이 데이터를 수집합니까?"** → "예" 선택.

### 4개 데이터 카테고리 추가

각 카테고리 공통 옵션:
- ☑ 연결되지 않음 (Not linked to user) — 익명 UUID, 신원 식별 불가
- ☑ 추적 안 함 (Not used for tracking) — 광고/제3자 공유 없음
- ☑ 사용 목적: **앱 기능 (App Functionality)** + **분석 (Analytics)**

| # | Apple 카테고리 경로 | 수집 내용 | 우리 구현 |
|---|---|---|---|
| 1 | 사용자 콘텐츠 → **기타 사용자 콘텐츠** | AI 튜터와의 대화 (사용자 입력 + AI 응답) | `codestudy_log.raw.userInput`, `aiOutput` |
| 2 | 식별자 → **디바이스 ID** | 익명 UUID (앱 첫 실행 시 생성, Apple IDFA 무관, 재설치 시 리셋) | `AnonymousID.current` |
| 3 | 사용 데이터 → **제품 상호 작용** | 마스터리 진행도, 세션 카운트, 스트릭, 학습 개념 ID | `concept_id`, `event='turn'`, `mastered` |
| 4 | 진단 → **성능 데이터** | 응답 지연 시간, 사용 AI 모델, token usage | `latencyMs`, `model`, `prompt_tokens`, `completion_tokens` |

### 보존 기간 / 처리

"앱 개인정보 보호 질문" 답변 시:
- 30일 이내 자동 삭제 명시 (PRIVACY.md와 일치)
- 광고·마케팅 사용 X
- 제3자 판매·공유 X

### 수집하지 않는 데이터 (명시)

위치, 연락처, 이메일, 사진(저장만), 결제 정보, 헬스 데이터, 검색 기록 등 일체 X.

`PrivacyInfo.xcprivacy` (이미 추가됨) + `PRIVACY.md` 4조 + 이 입력이 **3가지 일관**되어야 함.

| # | 화면 | Hook (한국어) | 노리는 페르소나 |

| 2 | **Swift 학습 채팅** (옵셔널 또는 SwiftUI 상태 같은 친숙 개념 진행 중) | `Swift, 답이 아닌 질문으로.` | iOS 페르소나 |

| 5 | **마스터리 완료** (confetti + streak badge) | `마스터 = 진짜로 이해했을 때.` | 양쪽 — 보상 시그널 |

**왜 이 순서**:
- **1번**: 양쪽 다 노출. iOS 개발자가 봐도 "내 거" / 백엔드 개발자가 봐도 "내 거" → 다음 슬라이드로 넘기는 의향 ↑
- **2-3번**: 페르소나 분기 노출. iOS 개발자는 2번에서 멈춰서 봄, 백엔드 개발자는 3번까지 넘김. 둘 다 "이거 내 stack이네" 확신
- **4번**: 진척 시각화로 "꾸준히 쓸 수 있는 도구" 시그널
- **5번**: 클로저. 마스터 = 진짜로 이해했을 때 보상이라는 차별화 메시지

## 톤 가이드

- **Hook은 punch** — 첫 3줄은 ~다 형태로 짧게 끊어 ("답을 준다. 잊힌다."). Indie 앱의 인상적 시작.
- **본문은 ~합니다 / ~합니다 격식** — 진지한 학습 도구 톤
- **Feature bullet은 짧은 명사구** — "광고 없음. 트래커 없음." 식으로 ㄴ다체와 명사형 혼용
- **Marketing 클리셰 금지** — "혁신적인", "획기적인", "최고의" 같은 단어 안 씀
- **숫자/구체 표현 우선** — "50개 이상의 Swift 개념", "5분이면 첫 개념" 같이 명확
- **이모지는 최소** — 새로운 기능에만 1-2개 OK, 설명 본문엔 X
