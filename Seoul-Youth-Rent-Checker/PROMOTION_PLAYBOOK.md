## 핵심 전제

**Viral coefficient (K-factor) 최적화가 전부.**
한 사용자가 평균 몇 명을 데려오느냐. K > 1이면 자생적 성장.
Tabber 케이스: 솔로 빌더 + 광고 0원 = 오로지 자연 공유에 의존.

**도구 공유 vs 결과 공유:**
- ❌ "내가 자격 됐어" 공유 (자랑 욕구 약함, 프라이버시 X)
- ✅ "친구야 너도 한 번 해봐" 공유 (도움 욕구, 사회적 자본 적립)

→ 모든 채널/콘텐츠는 "친구한테 알려줄 가치 있다" 톤.

## Viral loop 메커니즘

### 현재 (v1):
- 결과 페이지 → "친구에게도 공유하기" 버튼
- 공유 시 도구 URL만 (결과 X)
- localStorage + KV에 옵트인 이메일 저장

### 가능하면 (v1.5, 트래픽 보고):
- **referral counter** — "이미 N명이 진단했어요" 결과 페이지 표시
- **카운트다운 위젯** — 트위터 임베드 (Vercel Edge로 svg 카운트 발송)
- **마감 임박 카톡 알림** (5/18 cron — 이미 옵트인된 yo7504@kakao.com 등)

K-factor 1.0 목표:
- 결과 페이지 도달자 5%+ 가 "친구에게 공유하기" 클릭
- 공유받은 친구 30%+ 가 폼 시작
- → 공유 1회당 0.05 × 0.30 = 0.015. 100명이 공유하면 1.5명. 한참 부족 → **바이럴은 못 기대, 직접 공유가 답**.
